Архитектура проекта
Данный документ описывает программную архитектуру системы, предназначенной для мониторинга телеметрии автомобиля из Assetto Corsa и анализа состояния водителя.

1. Общая структура системы
Проект реализован на языке C++17 и разделен на логические модули, взаимодействующие в многопоточном режиме для обеспечения обработки данных с частотой не менее 30 FPS.

2. Основные компоненты (Модули)
2.1. Модуль телеметрии (OBD)
Класс OBDParser: Читает CSV-датасеты, полученные из Assetto Corsa. Выполняет валидацию данных (скорость, обороты, положение педалей) и преобразует их в структуру TelemetryData.
Класс ONNXClassifier: Загружает обученную модель driver_classifier.onnx. Выполняет нормализацию входных признаков (z-score) и классифицирует стиль вождения на три типа: Normal, Aggressive, Slow.

2.2. Модуль мониторинга водителя (DMS)
Класс DMSMonitor: Использует нейросетевой детектор лиц (Caffe) и каскады Хаара для поиска глаз.
Логика состояний: Анализирует степень открытости глаз и угол поворота головы. Реализует алгоритм «скользящего окна» (15 кадров) для определения усталости.

2.3. Модуль визуализации (UI)
Класс Dashboard: Отрисовывает динамические приборы (спидометр, тахометр) и полосы прогресса (топливо, температура) средствами OpenCV.
Класс DMSHUD: Накладывает рамки детекции и текстовые предупреждения (ALERTS) на видеопоток с веб-камеры.

3. Многопоточность и синхронизация
Для исключения задержек интерфейса при тяжелых вычислениях проект использует:
Thread 1 (Data/AI): Цикл обработки телеметрии и запуск модели классификации.
Thread 2 (Main/Graphics): Опрос веб-камеры, отрисовка GUI и вывод итогового окна.
SharedState: Общая структура данных, защищенная std::mutex для безопасной передачи телеметрии между потоками.

4. Стек технологий
OpenCV 4.x: Обработка изображений и графический интерфейс.
ONNX Runtime: Выполнение глубоких нейронных сетей.
Google Test: Автоматизированное тестирование парсера и логики алертов.
CMake: Кроссплатформенная система сборки.
